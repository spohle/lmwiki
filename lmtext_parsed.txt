This involves several distinct stages:\n*   **Tokenization:** Training custom tokenizers (e.g., using Rust implementations).\n*   **Pretraining:** Initial training on large corpora like FineWeb.\n*   **Fine-Tuning & Alignment:** Subsequent stages such as Midtraining on conversational data (like SmolTalk), Supervised Fine-Tuning (SFT) against benchmarks (MMLU, GSM8K, HumanEval), and optional Reinforcement Learning (RL) using methods like GRPO.\n\nThe goal of these minimal stacks is to provide a 'strong baseline' that is highly readable and forkable, allowing for rapid iteration on [[LLM Architecture]] and training methodologies.\n\n### 🛠️ Inference and Deployment\nOnce trained, the model must be made usable. This involves:\n*   **Efficient Inference:** Implementing optimized engines featuring techniques like KV caching for fast prefill/decode cycles.\n*   **Tool Use:** Integrating capabilities such as a Python interpreter within a lightweight sandbox to allow the LLM to interact with external tools.\n*   **User Interface:** Deploying the model via various interfaces, including CLI or web UIs (like a ChatGPT clone).\n\n### 📚 Knowledge Management and Application (LLM Knowledge Bases)\nBeyond pure training, LLMs are being leveraged as powerful knowledge processors. The concept of an [[LLM Knowledge Base]] involves using an LLM agent to manage and synthesize information from disparate sources:\n*   **Data Ingest:** Indexing raw data (papers, articles, repos) into a system.\n*   **Compilation & Structuring:** Using the LLM to incrementally 'compile' this data into a structured wiki format (a collection of `.md` files), automatically generating summaries and backlinks.\n*   **Interaction:** Allowing users to query the entire knowledge base via an agent, which researches answers against the indexed documents. This system can be viewed in IDEs like [[Obsidian]].\n*   **Maintenance & Refinement:** The LLM actively performs 'linting'—checking for data inconsistencies or suggesting new research paths—to maintain the integrity of the knowledge base.\n\nIn essence, these projects showcase a dual path: building smaller, understandable models from scratch, and using powerful LLMs as automated agents to build and manage complex personal knowledge systems."
