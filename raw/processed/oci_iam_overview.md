# OCI Identity and Access Management (IAM) Overview

## What is OCI IAM?
Identity and Access Management (IAM) is the service responsible for fine-grained access control and role-based access control (RBAC) within Oracle Cloud Infrastructure. It ensures that only authorized users can interact with specific cloud resources.

## Core Concepts: AuthN and AuthZ
The service is built on two foundational pillars:
* **Authentication (AuthN)**: Focuses on **Identity**. It verifies who someone is and ensures they are who they claim to be.
* **Authorization (AuthZ)**: Focuses on **Permissions**. It determines what a user is allowed to do by assigning them to roles with predetermined sets of permissions.

---

## Identity Domains
An **Identity Domain** acts as a container for users and groups. It represents a specific user population in OCI along with their associated security settings and configurations.

### How it Works in Practice:
1.  **Creation**: An identity domain is established.
2.  **Organization**: Users and groups are created within that domain.
3.  **Authentication**: Users verify their identity using common mechanisms like usernames and passwords.
4.  **Authorization**: Administrators write **Policies** against groups. These policies are scoped to either a **Tenancy** (the entire account) or a **Compartment** (a logical isolation for resources).

## Oracle Cloud Identifiers (OCID)
Every object created in OCI—such as a database, compute instance, or block volume—is considered a **Resource**. To manage these, Oracle automatically assigns each resource a unique identifier called an **OCID**.

### OCID Syntax
While the management console handles these in the background, you will interact with OCIDs frequently when using the Command Line Interface (CLI) or Software Development Kits (SDKs). The general syntax is:

`ocid1.<RESOURCE TYPE>.<REALM>.<REGION>.<UNIQUE ID>`

* **ocid1**: The version of the OCID.
* **Resource Type**: The type of resource (e.g., instance, volume, tenancy).
* **Realm**: A set of regions that share characteristics (e.g., commercial or government).
* **Region**: The code for the region where the resource resides (global resources like a Tenancy may omit this).
* **Unique ID**: A unique string generated by Oracle to identify that specific resource.

## Summary
* **Authentication** = Identity (Who are you?)
* **Authorization** = Permissions (What can you do?)
* **OCID** = The unique "fingerprint" for every resource in the cloud.
