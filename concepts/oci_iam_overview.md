# OCI Identity and Access Management (IAM) Overview

[[Oracle Cloud Infrastructure]] (OCI) [[Identity and Access Management]] (IAM) is a critical service for managing [[fine-grained access control]] and [[role-based access control]] (RBAC) within the OCI environment. Its primary function is to ensure that only authorized users and services can interact with specific cloud resources.

## Core Concepts: Authentication (AuthN) and Authorization (AuthZ)
IAM is built upon two fundamental pillars:
*   **[[Authentication]] (AuthN)**: This process verifies the **identity** of a user or service, confirming "who someone is" and that they are legitimate.
*   **[[Authorization]] (AuthZ)**: This process determines **permissions**, defining "what a user is allowed to do" based on assigned roles and policies.

## Identity Domains
An [[Identity Domain]] serves as a logical container for users and groups within OCI. It encapsulates a specific user population, along with their associated security settings and configurations.

### How Identity Domains Work:
1.  **Creation**: An identity domain is established within OCI.
2.  **Organization**: [[Users]] and [[Groups]] are created and managed within this domain.
3.  **Authentication**: Users verify their identity, typically using mechanisms like usernames and passwords.
4.  **Authorization**: Administrators define [[Policies]] that grant specific permissions to groups. These policies can be scoped to either the entire [[Tenancy]] (the root account) or a specific [[Compartment]] (a logical isolation unit for resources).

## Oracle Cloud Identifiers (OCID)
Every object or service created in OCI, such as a database, compute instance, or block volume, is considered a [[Resource]]. To uniquely identify and manage these resources, Oracle automatically assigns each one an [[Oracle Cloud Identifier]] (OCID).

### OCID Syntax
While the OCI Console often abstracts these, OCIDs are frequently used when interacting with the [[Command Line Interface]] (CLI) or [[Software Development Kits]] (SDKs). The general syntax is:

`ocid1.<RESOURCE TYPE>.<REALM>.<REGION>.<UNIQUE ID>`

*   **ocid1**: Indicates the version of the OCID format.
*   **Resource Type**: Specifies the type of resource (e.g., `instance`, `volume`, `tenancy`).
*   **Realm**: Denotes a set of regions sharing common characteristics (e.g., commercial or government).
*   **Region**: The code for the specific region where the resource resides (global resources like a Tenancy may omit this).
*   **Unique ID**: A unique string generated by Oracle to identify that specific resource.

## Summary
*   **Authentication** = Identity (Who are you?)
*   **Authorization** = Permissions (What can you do?)
*   **OCID** = The unique "fingerprint" for every resource in the cloud.
